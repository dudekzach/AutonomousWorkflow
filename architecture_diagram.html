Layer 1 — User Experience

Hosted UI used by solution consultants and internal teammates

Web UI

Users open a public Render URL and interact through a custom HTML interface instead of a terminal.

Prompt Input

Users enter a goal-oriented prompt with context, audience, tone, and output expectations.

Optional File Attachments

Users can upload supporting files such as notes, transcripts, drafts, or exported text-based artifacts.

Run Controls

Current UX supports core execution. Near-term controls can expose optimizer strategy, target model, tone, and format preferences.

→

Layer 2 — API & App Layer

FastAPI service hosted on Render

FastAPI Routes

/ serves the UI, /run handles normal execution, /run-with-files accepts multipart form uploads, and /outputs serves generated artifacts.

Request Packaging

Prompt, user context, iteration count, options, and uploaded file content are normalized into a single execution payload.

Deployment & Sharing

Render provides a stable public URL so the workflow can be shared without exposing a local laptop.

Secrets Management

API keys are stored as environment variables in the host environment instead of the repository.

→

Layer 3 — Prompt Engineering Layer

New intelligence before model execution

Prompt Optimizer

Before main execution, the user prompt is refined for clarity, specificity, structure, efficiency, and creativity control.

  • Produces optimized prompt
  • Can compare optimizer candidates
  • Can store optimizer rationale / metadata

Attachment Context Injection

Uploaded file contents are appended into a structured prompt context block so downstream models can use supporting material.

Normalized Active Prompt

The optimized prompt becomes the actual prompt sent to the execution models.

→

Layer 4 — Model Execution Layer

Parallel answer generation across multiple providers

OpenAI Path

Sends the optimized prompt to the OpenAI API and captures a first-pass response plus response state for follow-ups.

Claude Path

Sends the same optimized prompt to Anthropic and captures a first-pass response plus conversation state for follow-ups.

Same Prompt, Independent Reasoning

The architecture intentionally compares how different frontier models solve the same business problem.

→

Layer 5 — Evaluation & Orchestration

Decisioning turns raw outputs into a guided workflow

Judge Layer

A judge model scores both outputs on clarity, completeness, tone, practical usefulness, and overall strength.

Autonomous Next Action

The system decides whether to accept, revise and rerun, or follow up on the stronger response.

Continuation Handling

If a winning answer is cut off, the workflow triggers continuation prompts until the answer is sufficiently complete.

Stitching Step

When needed, original and continuation outputs are merged into one polished final answer instead of exposing a broken draft plus patch.

→

Layer 6 — Output & Business Value

What the user sees and what the business gains

UI Result Panel

Displays status, runtime, summary, final output, logs, and generated artifact links.

Artifacts

Each run generates structured JSON and an HTML report for review, sharing, and replayability.

Traceability

Artifacts preserve prompt, optimized prompt, model outputs, judge reasoning, and final recommendation.

Reusable Internal Tooling

This creates a foundation for repeatable demo prep, defense strategy generation, evaluation workflows, and future agentic automation.
