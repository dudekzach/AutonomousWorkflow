Iteration Request Lifecycle

  1. Ingestion: Prompt + Files received via /run-with-files.
  2. Optimization: System applies FORCE_OPTIMIZER_OFF check and triggers prompt refinement.
  3. Competitive Execution: Parallel calls to OpenAI GPT-4o and Claude 3.5 Sonnet.
  4. The Judge Decision: OpenAI-based judge scores both outputs on clarity, completeness, and tone.
  5. Branching Logic:
    • accept: Finalizes the workflow.
    • revise_prompt_and_rerun: Triggers next iteration loop.
    • follow_up_on_winner: Deep-dives into the winning response.
  6. Finalization: Optional stitching of multi-turn history into a single cohesive artifact.

Environment & Dependencies

  • Web Framework FastAPI / Uvicorn
  • Model A (Primary) OpenAI (gpt-4.1-mini)
  • Model B (Challenger) Anthropic (claude-sonnet-4-6)
  • Persistence Redis (redis-py)
  • Client Side Pure HTML5 / Vanilla JS
  • Security python-dotenv / Truststore
SC Architect Note: This architecture is optimized for high-trust executive demos. It utilizes a "competitive grounding" approach where the judge model acts as an internal quality controller before any artifact is presented to the user.
